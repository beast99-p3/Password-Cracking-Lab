Rule-Based Attack

Applies mutation rules to each wordlist entry — capitalise, append numbers, leet speak, etc.

Lesson
Users predictably mutate passwords: capitalise the first letter, swap a→@, append a year. Rule-based attacks exploit these patterns, cracking passwords that survive a plain dictionary attack.

Credential Stuffing Attack

Using the cracked MegaBank XYZ credentials, an attacker automatically tries the same (or similar) passwords on 8 popular external services. This works because 65 %+ of users reuse passwords across sites.

How Credential Stuffing Works
  1. Attacker buys/steals a leaked credentials database
  2. Automates login attempts across hundreds of services
  3. Even a 1–3 % success rate yields thousands of accounts
  4. Compromised accounts are sold, used for fraud, or ransomed
How to Defend
  • Never reuse passwords — use a password manager
  • Enable 2FA / MFA on every account
  • Services: implement rate limiting + CAPTCHA
  • Check accounts on haveibeenpwned.com regularly
